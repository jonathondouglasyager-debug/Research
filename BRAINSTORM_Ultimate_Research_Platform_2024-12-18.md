## üìä REQUIRED DATA SOURCES & APIs

### Essential (High Priority)
1. **SEC EDGAR Database**
   - Type: Free API
   - Contains: 10-K, 10-Q, 8-K filings, proxy statements, ownership changes
   - Reveals: Financial history, executive compensation, board changes

2. **Flight Tracking Data**
   - Sources: FlightRadar24 API, ADS-B Exchange
   - Type: Paid subscription
   - Contains: Private jet movements, tail numbers, flight histories
   - Reveals: Executive meetings, undisclosed travel, timeline context

3. **Court Records (PACER)**
   - Type: Paid per document
   - Contains: Federal court cases, settlements, legal disputes
   - Reveals: Hidden agreements, trust modifications, corporate battles

4. **Political Data (OpenSecrets)**
   - Type: Free API
   - Contains: Campaign contributions, lobbying expenditures, FEC data
   - Reveals: Political influence, regulatory relationships

5. **Corporate Registrations**
   - Sources: State business registry APIs (varies by state)
   - Type: Mostly free, some paid
   - Contains: Entity formations, DBA names, registered agents
   - Reveals: Corporate structures, shell companies, subsidiaries

### Important (Medium Priority)
6. **News Archives**
   - Sources: NewsAPI, LexisNexis, historical archives
   - Type: Paid subscriptions
   - Contains: All published news articles
   - Reveals: Public narrative, timeline context

7. **FOIA Document Databases**
   - Sources: MuckRock, DocumentCloud, agency portals
   - Type: Free (mostly)
   - Contains: Previously released government documents
   - Reveals: Government interactions, regulatory correspondence

8. **Patent Database (USPTO)**
   - Type: Free
   - Contains: Patent filings, inventors, assignees
   - Reveals: Technology connections, corporate R&D

### Supporting Sources
9. **Academic Databases**
   - PubMed (medical/scientific research)
   - arXiv (preprints)
   - Google Scholar (cross-discipline)

10. **Social Media Archives**
    - Twitter/X historical data
    - LinkedIn corporate connections
    - Public social graphs

   - Start with free sources only?

2. **Data storage:** Where/how do we store retrieved documents?

   - Hybrid approach?

3. **Processing power:** Some analysis is computationally intensive

   - Batch vs real-time?

1. **Starting point:** Do we build Fox research end-to-end first, or build modular components?

2. **User interface:** Command-line tool, web app, desktop app, or all three?

3. **Monetization:** When/how do we think about revenue?
   - Free tier vs paid?
   - Subscription model?
   - API access fees?

### Philosophical
1. **Scope creep:** How do we avoid building "everything for everyone"?
