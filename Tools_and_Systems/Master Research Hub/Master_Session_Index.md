## Session Naming Convention

**Format:** `RS-YYYYMMDD-###`

- **RS** = Research Session
- **YYYYMMDD** = Date (20241202 = December 2, 2024)
- **###** = Session number that day (001, 002, etc.)

**Example:** RS-20241202-001 = First research session on December 2, 2024

## Integration with Existing Databases

### When a research session connects to existing work:
1. Note the connection in the session file
2. Add cross-reference here in Master Index
3. Update the existing database with new findings
4. Link both directions (session → database, database → session)

### Example Integration Points:
- **Geoengineering Legislation Database** → Link relevant sessions
- **Flight Tracking Infrastructure** → Cross-reference aircraft/contractor data
- **Contractor Network Maps** → Add new entity connections
- **Weather Intelligence Platforms** → Update with new sources

## Notes & Methodology Reminders

- **Truth-seeking principle:** We only seek truth, we only tell truth
- **Permission to be wrong:** Better to say "I don't know" than speculate
- **Primary sources prioritized:** Always chase the original document
- **Evidence matrix thinking:** What supports AND what contradicts
- **Explicit dot-connecting:** Don't assume - document the connection
- **Source quality over quantity:** One excellent source beats ten mediocre ones
- **Systematic skepticism:** Actively seek disconfirmation
